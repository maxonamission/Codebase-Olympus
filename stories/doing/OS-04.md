# Story OS-04: CI baseline (GitHub Actions)

## Doel
Een GitHub Actions-workflow opzetten die op elke push en pull request automatisch de kwaliteitschecks draait op een schone server. Dit is laag 4 van de ontwikkelstraat: de harde poort. Pre-commit kun je lokaal omzeilen met `--no-verify`; CI kun je niet omzeilen.

## Input
- Ruff-configuratie uit OS-01
- Mypy-configuratie uit OS-02
- Bestaande pytest-suite (165 tests)
- `pyproject.toml` met `uv`-compatible dependency-groep `dev`

## Acceptatiecriteria
- [x] `.github/workflows/ci.yml` op repo-root
- [x] Triggert op: `push` (alle branches) en `pull_request` naar `main`
- [x] Python 3.11 matrix (uitbreidbaar naar 3.12 later)
- [x] Stappen in deze volgorde:
  1. Checkout (actions/checkout@v5)
  2. Setup uv + Python 3.11 (astral-sh/setup-uv@v7, met `enable-cache: true`)
  3. `uv pip install --system -e ".[dev]"`
  4. `uv run ruff check .`
  5. `uv run ruff format --check .`
  6. `uv run mypy src/`
  7. `uv run pytest --cov=src --cov-report=term-missing --cov-report=xml`
  8. Upload coverage als GitHub Actions-artefact (actions/upload-artifact@v4, retention 14 dagen)
- [x] `timeout-minutes: 10` + `concurrency`-group met `cancel-in-progress` ter voorkoming van wachtrij-opbouw; lokaal gesimuleerd draait de hele suite in ~26s → ruim onder 3 minuten op GitHub-runners
- [x] `uv`-cache actief via `enable-cache: true` + `cache-dependency-glob: pyproject.toml`
- [x] Status-badge toegevoegd aan `README.md`
- [x] `.gitignore` uitgebreid met `coverage.xml`, `.mypy_cache/`, `.ruff_cache/`
- [ ] Branch protection op `main` geactiveerd met CI als vereiste check — **kan niet vanuit CLI**, vereist GitHub-UI; genoteerd als follow-up voor de user
- [ ] Een break-test met falende test wordt geblokkeerd — **kan niet vanuit deze sessie**, vereist daadwerkelijke CI-run op GitHub; genoteerd als follow-up
- [ ] Een break-test met lint-fout wordt geblokkeerd — **idem**, CI-run nodig
- [x] Lokale CI-simulatie groen: ruff check + ruff format + mypy + pytest (558 tests, 94% coverage)

## Scope
Eén workflow, één job, alle checks sequentieel. Parallelisering (matrix per check) is optimalisatie voor later als dit te langzaam wordt.

## Niet in scope
- Deploy-workflow → later, hoort bij E4-deployment-epic
- Cross-OS-matrix (Windows/macOS) — nvt, Olympus draait alleen op Linux-server
- Coverage-thresholds die builds blokkeren — alleen rapporteren; optrekken naar threshold is latere story
- Release-automation (versioning, PyPI)
- Storystatus/AC-check → OS-06 (wordt als extra step toegevoegd daar)

## Aanpak
1. `.github/workflows/ci.yml` schrijven
2. Eerst pushen op feature-branch, debuggen tot groen
3. Break-tests uitvoeren en resultaten noteren
4. README-badge toevoegen
5. Branch protection configureren op GitHub (repo-settings)

## Geschat
1 middag. Risico: `uv` in GitHub Actions heeft caveats (cache-keys, wheel-compatibility). Mitigatie: beginnen met `setup-uv@v3` van Astral, dat is het meest voorspelbaar.

## Resultaat
- `.github/workflows/ci.yml` geschreven met één `checks`-job, sequentiële stappen, matrix Python 3.11
- `astral-sh/setup-uv@v7` gekozen voor dependency-cache (nieuwere versie dan AC suggereerde — functioneel gelijk)
- `concurrency`-block: nieuwe push op dezelfde branch annuleert oudere runs
- Lokale CI-simulatie: alle stappen groen in ~26s (558 tests, 94% totale coverage)
- README heeft badge die naar de CI-run verwijst
- `.gitignore` uitgebreid voor CI-artefacten

## Follow-up voor de user (niet vanuit deze sessie uitvoerbaar)
1. **Branch protection op `main` activeren** in GitHub-repo-settings → Rules → "Require status checks" → selecteer `checks / CI (3.11)`
2. **Eerste CI-run verifiëren** door deze PR (of een andere) te openen en te checken of de workflow draait en groen is
3. **Break-test uitvoeren** op een wegwerp-branch: push een opzettelijke faling en verifieer dat CI blokkeert
4. Zodra 1-3 gedaan: deze story heeft nog één open AC-items die met een follow-up-commit afgevinkt kunnen worden, of kan geaccepteerd worden als "infrastructuur staat; effectiviteit wacht op eerste echte PR"
