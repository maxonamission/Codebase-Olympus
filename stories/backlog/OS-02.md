# Story OS-02: Type-checking toevoegen (mypy)

## Doel
Statische type-checking invoeren zodat fouten in types vroeg gevangen worden, niet pas bij runtime. De codebase gebruikt al type hints overal (CLAUDE.md vereist dat) en Pydantic; mypy maakt die hints afdwingbaar. Baseline-aanpak: nieuwe code moet schoon zijn, bestaande fouten worden geaccepteerd maar niet meer uitgebreid.

## Input
- Bestaande Python-code met type hints in `src/gymnasium_classica/`, `scripts/`, `tests/`
- CLAUDE.md-regel: "Gebruik type hints overal. Pydantic models voor data validation."
- Huidige `pyproject.toml` zonder mypy-config

## Acceptatiecriteria
- [ ] `mypy` toegevoegd aan `[project.optional-dependencies].dev` in `pyproject.toml`
- [ ] `[tool.mypy]` sectie in `pyproject.toml` met: `python_version = "3.11"`, `strict = true`, `exclude` voor scripts die we nog niet strict kunnen maken
- [ ] `[[tool.mypy.overrides]]` per module voor bekende zachte gevallen (bv. externe libs zonder stubs)
- [ ] `uv run mypy src/` draait succesvol (al dan niet met baseline)
- [ ] Indien error-baseline nodig: bestand `mypy_baseline.txt` met huidige errors, script om nieuwe errors tegen baseline te checken
- [ ] Pydantic-plugin ingeschakeld: `plugins = ["pydantic.mypy"]`
- [ ] Commando gedocumenteerd in CLAUDE.md of README: `uv run mypy src/`
- [ ] Alle bestaande 165 tests blijven groen
- [ ] Als `mypy src/` nu groen kan worden met auto-fixes: doen. Zo niet: baseline vastleggen.

## Scope
Configuratie + baseline-vaststelling. Eventuele triviale type-fixes (bv. `list` → `list[str]`, Optional-annotations toevoegen waar nodig).

## Niet in scope
- Pyright overwegen (alleen mypy voor nu, kiezen we later eventueel opnieuw)
- `tests/` onder strict mypy brengen (test-mocks maken dat vaak onhaalbaar; apart override-block)
- Alle bestaande mypy-errors wegpoetsen — dat wordt een apart project ("tech-debt")
- Pre-commit integratie → OS-03
- CI integratie → OS-04

## Aanpak
1. `mypy` + `pydantic` plugin toevoegen
2. `pyproject.toml` configuratie schrijven, strict aan
3. `uv run mypy src/` draaien; output beoordelen
4. Als <20 errors: handmatig fixen. Als meer: baseline-bestand genereren met `mypy src/ | tee mypy_baseline.txt`
5. Kort script `scripts/check_mypy_no_regression.py` dat current mypy-output vergelijkt met baseline (alleen nieuwe errors blokkeren). Optioneel — mag ook naar OS-04/CI verschoven worden.
6. Documentatie-regel

## Geschat
1 middag. Risico: Pydantic v2 + generics in `models/` leveren mogelijk complexe errors. Mitigatie: tijdelijk `warn_unused_ignores = false` en module-level `# type: ignore[misc]` met TODO.
