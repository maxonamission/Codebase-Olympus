# Story OS-05: Claude Code hooks (PostToolUse + Stop)

## Doel
Via `.claude/settings.json` automatische checks laten draaien binnen Claude Code-sessies, zodat Claude fouten meteen ziet en in dezelfde sessie oplost — niet pas nadat een PR is aangemaakt. Dit is laag 3 van de ontwikkelstraat. Waarde: reparaties gebeuren binnen de context waarin Claude nog weet wat hij aan het doen was.

## Input
- Ruff-configuratie uit OS-01
- Mypy-configuratie uit OS-02
- Bestaande pytest-suite
- Claude Code hooks-documentatie (PostToolUse, Stop, SessionStart)
- Skill `update-config` in Claude Code kan helpen bij correcte syntax

## Acceptatiecriteria
- [ ] `.claude/settings.json` op repo-root met minimaal deze hooks:
  - [ ] **PostToolUse** op `Edit|Write` voor `*.py`: draait `uv run ruff check --fix` + `uv run ruff format` op het gewijzigde bestand
  - [ ] **Stop**: draait `uv run pytest -x --tb=short` (snelle modus, stopt bij eerste failure)
- [ ] Hook-scripts zijn idempotent en snel (PostToolUse onder 2s op één bestand)
- [ ] Hooks falen niet-fataal bij tooling-fout (ruff niet gevonden → waarschuwing, geen crash)
- [ ] Hooks werken met `uv` als package manager (niet globale pip)
- [ ] Een voorbeeldsessie getest: Claude schrijft opzettelijk ongeldige code → PostToolUse rapporteert → Claude corrigeert in dezelfde turn
- [ ] Een voorbeeldsessie getest: Claude probeert te stoppen terwijl een test faalt → Stop-hook blokkeert → Claude fixt
- [ ] `.claude/settings.json` staat in versiebeheer; eventuele lokale overrides in `.claude/settings.local.json` (gitignored)
- [ ] Instructie in CLAUDE.md toegevoegd: welke hooks actief zijn en wat ze doen

## Scope
Alleen `PostToolUse` en `Stop`. Andere hook-typen (SessionStart, UserPromptSubmit) zijn uitbreidingen voor latere stories.

## Niet in scope
- SessionStart-hook voor venv-setup — interessant, maar vereist dat elke developer dezelfde `uv`-install heeft; apart afwegen
- Hook voor story-status → OS-06 (wordt daar eventueel ook als Stop-hook toegevoegd)
- Hooks voor non-Python bestanden (als later React-frontend bijkomt)
- MCP-server-configuratie

## Aanpak
1. `.claude/settings.json` schrijven met hooks-block
2. Shell-snippets voor elk hook (of verwijzen naar scripts in `scripts/hooks/`)
3. Handmatig testen: edit-een-py-bestand-met-fout scenario
4. Handmatig testen: opzettelijk gebroken-test scenario
5. Documentatie

## Geschat
Halve middag. Risico: `PostToolUse` hook-output verstoort Claude's output-stream als niet goed ingesteld. Mitigatie: altijd zachte exit-code 0 tenzij blocking gewenst, output beperken tot relevante regels.
