# Story OS-08: CLAUDE.md bijwerken met ontwikkelstraat-sectie

## Doel
De volledige ontwikkelstraat zoals die in OS-01 t/m OS-07 is opgebouwd, documenteren in `CLAUDE.md` zodat het de **gedeelde standaard** wordt voor zowel mens als AI. Dit is laag 6: één bron van waarheid. Zonder deze documentatie verdampt de straat na een paar maanden — niemand weet meer wat de regels zijn, en drift begint.

## Input
- Alle voorgaande OS-stories in `done/`
- Bestaande CLAUDE.md (7,7 KB, goed opgezet)
- `docs/ontwikkelstraat-uitleg.md` als achtergrondmateriaal

## Acceptatiecriteria
- [ ] Nieuwe sectie `## Ontwikkelstraat` in CLAUDE.md, na `## Werkwijze`
- [ ] Per laag een korte subsectie met:
  - [ ] Wat wordt gecheckt
  - [ ] Wanneer (lokaal / op commit / in CI / in sessie)
  - [ ] Welk commando de ontwikkelaar zelf kan draaien
  - [ ] Wat te doen bij een faling
- [ ] Subsecties minimaal:
  - [ ] Lint & format (ruff)
  - [ ] Type-checking (mypy)
  - [ ] Pre-commit hooks
  - [ ] CI-workflow
  - [ ] Claude Code hooks
  - [ ] Story-status en AC-verificatie
  - [ ] Review-skills
- [ ] Sectie `## Story-workflow` die beschrijft:
  - [ ] Wanneer een story naar `doing/` verhuist
  - [ ] Wanneer naar `done/`
  - [ ] Wanneer `EPICS.md` bijgewerkt wordt
  - [ ] Hoe AC afvinken (niet alleen in bestand, ook consistent houden)
- [ ] Korte verwijzing naar `docs/ontwikkelstraat-uitleg.md` voor wie achtergrond wil
- [ ] "Session-management"-sectie in CLAUDE.md bijgewerkt met lessons uit OS-05 (Claude-hooks) — bv. Stop-hook draait pytest, dus niet onverwacht hangen
- [ ] Break-testen: bij een fictieve faling weet Claude welke stap hij moet zetten (uittesten met een bewust gebroken scenario in een voorbeeldsessie)
- [ ] Alle bestaande stories, scripts en tests blijven ongewijzigd

## Scope
Alleen documentatie. Geen technische wijzigingen. Wel: als tijdens het schrijven blijkt dat een eerdere story een gat heeft (bv. een onbegrijpelijke foutmelding uit OS-03), noteren als follow-up — niet stilletjes fixen in deze story.

## Niet in scope
- Uitrol naar andere projecten (voxtral-transcribe, toekomstige apps) — aparte epic
- Template-repo `my-templates/` opzetten — aparte epic
- Engelse vertaling van CLAUDE.md — Nederlands blijft (conform project-conventie)
- Gedetailleerde troubleshooting-gids per tool (blijft beknopt; linker naar tool-docs voor diepte)

## Aanpak
1. Draft-sectie schrijven op basis van de feitelijke eindstand na OS-01 t/m OS-07
2. Per laag: één alinea + commando-voorbeeld
3. Story-workflow-sectie: stapsgewijs ("pak story, verplaats naar doing/, commit; bij klaar: AC afvinken, verplaats naar done/, werk EPICS.md bij, commit")
4. Review-ronde op eigen tekst (schrappen, korter)
5. `/review` draaien op de PR
6. Eventueel in een voorbeeldsessie Claude instructies laten volgen → verifiëren of de documentatie genoeg is

## Geschat
Halve middag. Grootste risico: sectie wordt te lang en niemand leest hem. Tegenmaatregel: harde limiet van 60 regels voor de hele ontwikkelstraat-sectie, met linker naar details in `docs/`.
