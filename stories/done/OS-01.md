# Story OS-01: Lint- en formatbasis afmaken (ruff strict)

## Doel
De bestaande ruff-configuratie uitbreiden van minimaal ("target-version" + "line-length") naar expliciet en streng, zodat `ruff check` en `ruff format --check` over de hele codebase groen zijn én naar meer code-kwaliteits-issues kijken dan de ruff-defaults. Dit is de fundering voor pre-commit (OS-03), CI (OS-04) en Claude Code hooks (OS-05).

## Input
- Huidige `pyproject.toml` (regels 33-35 met `[tool.ruff]`)
- Bestaande Python-code in `src/gymnasium_classica/`, `scripts/`, `tests/`
- CLAUDE.md coding-conventies (type hints overal, Pydantic models, docstrings/comments in Engels)

## Acceptatiecriteria
- [x] `[tool.ruff.lint]` sectie in `pyproject.toml` met expliciete `select`: E, F, I, UP, B, SIM, RUF
- [x] `[tool.ruff.lint.per-file-ignores]` voor tests (B011, E402) en scripts (B008, E402)
- [x] `[tool.ruff.format]` sectie expliciet (quote-style, indent-style, line-ending)
- [x] `ruff check .` geeft nul errors op de hele repo
- [x] `ruff format --check .` geeft nul diffs op de hele repo
- [x] CLAUDE.md vermeldt de commando's `uv run ruff check .` en `uv run ruff format .`
- [x] Regelkeuzes hebben een korte rationale-comment in `pyproject.toml` (incl. waarom RUF001-003 op ignore)
- [x] Alle bestaande tests blijven groen (558 passed — het getal in de originele story was achterhaald)
- [x] Functionele wijzigingen aan src beperkt tot wat ruff rapporteert: B904-chaining (`raise ... from err`) in 6 API-handlers, E741 noqa's waar enum/ID-schema dat dicteert, RUF034 simplificatie in `validate_items_c1_11.py` (dode if/else), RUF043 regex-raw-string fix

## Scope
Alleen configuratie + automatische fixes (`ruff check --fix`, `ruff format`). Handmatige refactoring alleen als ruff een fout rapporteert die niet auto-fixbaar is én de fout is echt (geen false positive). Bij false positives: regel onderdrukken met `# noqa: RULECODE` met toelichting.

## Niet in scope
- Type-checking (mypy/pyright) → OS-02
- Pre-commit integratie → OS-03
- CI integratie → OS-04
- Strenge regels zoals `D` (docstrings) of `ANN` (annotations) — die overwegen we later als OS-02 staat

## Aanpak
1. `pyproject.toml` uitbreiden met nieuwe lint-sectie + per-file-ignores
2. `uv run ruff check --fix .` draaien en resultaat reviewen
3. `uv run ruff format .` draaien
4. Testsuite draaien → moet groen blijven
5. Resterende manuele fixes (indien nodig) in één commit
6. Documentatie-regel toevoegen aan CLAUDE.md of README

## Geschat
1 middag werk. Grootste risico: bugbear (B) of simplify (SIM) rapporteert veel issues in bestaande code — dan scope beperken tot auto-fix en niet-triviale issues als apart "tech-debt"-lijstje opschrijven.

## Resultaat
- Startsituatie: 117 ruff-errors met defaults, 84 files unformatted
- Na uitgebreide regelkeuze: 737 errors (waarvan 464 RUF001 door Griekse tekens — false positive, op ignore)
- Na `ruff check --fix`: 179 errors opgelost (veel F401 unused imports, I001 imports-order, UP045 modern optional-syntax)
- Na `ruff check --fix --unsafe-fixes`: nog eens 54 opgelost (B007 unused loop vars, RUF059 unused unpacked vars, F841 unused vars)
- 16 resterende handmatig gefixt: 6× B904 (raise from), 4× E741 (noqa bij enum I), 2× SIM102 (collapsible if), 1× RUF034, 1× RUF043, 1× B017, 1× E741 rename
- Eindstand: `ruff check` groen, `ruff format` groen, 558 pytest-tests groen
- Commits op branch `claude/mobile-app-exploration-S1u5U`: `6c50591` (auto-fixes + format), finale commit bij afronding story
