# Story L2-03: Learner-niveau parameters (individuele leersnelheid)

## Doel
Voeg **learner-niveau parameters** toe naast de skill-niveau
BKT-parameters, te beginnen met een individuele leersnelheid, zodat het
model rekening houdt met verschillen tussen leerlingen.

## Achtergrond
Ontwerpkeuze 14. Bij (jonge) taalleerders wegen individuele verschillen
vaak zwaarder dan het exacte spacing-interval (Kasprowicz, 2019), en
geïndividualiseerde BKT (parametrisering van leersnelheid) verbetert de
voorspelling van ongeziene leerlingen aantoonbaar (Yudelson, 2013).

## Input
- `src/gymnasium_classica/models/learner.py` — `LearnerModel`,
  `KnoopState`, BKT-update
- L2-02 (strategie-interface) als aangrijpingspunt

## Acceptatiecriteria
- [x] `LearnerModel` krijgt minstens één learner-niveau parameter
      (individuele leersnelheid / transitie-modifier) met een neutrale
      default
- [x] De BKT-update gebruikt de learner-parameter bovenop de
      skill-parameters; default-waarde reproduceert het huidige gedrag
- [x] De parameter wordt geschat/bijgewerkt op basis van de
      observatiegeschiedenis (eenvoudige, interpreteerbare update)
- [x] Unit-tests: een leerling met hogere leersnelheid bereikt mastery
      sneller bij gelijke observaties; neutrale default = huidige uitkomst
- [x] Geen breaking changes; bestaande tests blijven groen

## Scope
Eén learner-niveau parameter en de schatting ervan. Geen volledige
hiërarchische Bayesiaanse herimplementatie.

## Afhankelijkheden
- L2-02 (strategie-interface) wenselijk als basis.

## Geschat
Medium.

## Resultaat
- `LearnerModel.learning_rate` (default 1.0, gt 0) — learner-niveau modifier
  op de BKT-transitie P(T), bovenop de skill-parameters.
- `bkt_update_posterior(..., learning_rate=1.0)`: `p_t = min(0.99,
  p_transit * learning_rate)`; 1.0 reproduceert het huidige gedrag exact.
  `update_node_state` geeft `learner.learning_rate` door (ook per richting).
- `estimate_learning_rate(learner)`: interpreteerbare schatting uit de
  observatiegeschiedenis — gemiddelde residu van uitkomst vs.
  `ItemResponse.mastery_before`, geclamped naar [0.5, 2.0]. Herschat in
  `_process_response` na elk vastgelegd antwoord.
- Tests: hogere leersnelheid → meer winst per observatie en mastery in
  minder stappen; out-/underperformer → rate >/< 1.0; clamping; neutrale
  default == baseline. **639 tests groen**, geen regressie.
