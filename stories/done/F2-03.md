# Story F2-03: Struikelpunten-overzicht per leerling

## Doel
Een mentor ziet in één oogopslag welke knopen voor een leerling
problematisch zijn: sorteerbaar op foutpercentage, op absolute
aantal-foute-pogingen en op recency.  Geen diepteklinkers meer;
per-item-details blijven in F2-02.

## Input
- `KnoopState.item_history` (F1-12)
- `KnoopState.posterior_mastery` voor kruisreferentie

## Acceptatiecriteria
- [x] Endpoint `GET /mentor/{user_id}/struikelpunten` retourneert per
      knoop: totaal-pogingen, foute-pogingen, foutpercentage, laatste-
      datum, huidige mastery.  Default gesorteerd op recente fouten.
- [x] Query-param `?min_attempts=3` om ruis te filteren
- [x] Frontend-tabel met sortable columns
- [x] Klikken op een rij linkt door naar F2-02-view

## Scope
- Alleen per-leerling (klas-aggregatie: optionele latere uitbreiding)
- Geen grafieken/heatmap — platte tabel

## Geschat
Medium

## Resultaat

**Backend:**
- `GET /mentor/{user_id}/struikelpunten?min_attempts=3` in
  `api/routes/mentor.py` (guard F2-01). Een knoop telt mee bij ≥1 foute
  poging én ≥ `min_attempts` totaal. Per knoop: totaal, fout, fout%,
  laatste foute datum, mastery. Default-sortering: meest recente fout
  eerst. Schemas `StruikelpuntEntry` / `StruikelpuntenResponse`.
- 7 pytest-tests (`tests/test_api_mentor_struikelpunten.py`): aggregatie,
  min_attempts-filter, foutloze knopen uitgesloten, default-sortering,
  leeg, 403 niet-gekoppeld, 403 leerling.

**Frontend:**
- `components/MentorStruikelpunten.jsx` — sorteerbare tabel (klik op
  kolomkop sorteert/toggelt; `aria-sort`). Rij-klik én Enter roepen
  `onSelect(knoop_id)` aan → parent linkt door naar de F2-02-pogingen.
- Pure sorteerhulp losgetrokken naar `components/struikelpuntSort.js`
  (testbaar + react-refresh-lintregel).
- `api.getStruikelpunten(userId, minAttempts)`; tabelstijlen in main.css.
- 7 vitest-tests: sorteerhulp (numeriek/lexicaal, niet-muterend) +
  component (leeg, rijen/percentages, onSelect, her-sorteren op kolom).

**Buiten scope (bewust):** klas-aggregatie en grafieken/heatmap (AC).
Suites: backend 726 groen, frontend 42 groen, mypy + ruff + eslint
(eigen bestanden) schoon.
