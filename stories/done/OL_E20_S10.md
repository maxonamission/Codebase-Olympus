# Story OL_E20_S10: Opschonen passages — lat_passages_leerjaar1.json mergen in lat_leespassages_leerjaar1.json

## Doel
In `data/passages/` staan twee Latijnse passage-bestanden naast elkaar:
- `lat_passages_leerjaar1.json` — 3 passages, oudere PoC-uitvoer
- `lat_leespassages_leerjaar1.json` — 20 passages, gebouwd in OL_E28_S3

De loader laadt beide directories-breed (`passages/loader.py:37-56`), wat werkt maar maakt de naamgeving inconsistent en bron-van-waarheid onduidelijk. Duplicatie-risico groeit wanneer nieuwe auteur/pensum-passages erbij komen.

## Input
- `data/passages/lat_passages_leerjaar1.json`
- `data/passages/lat_leespassages_leerjaar1.json`
- `src/gymnasium_classica/passages/loader.py`

## Acceptatiecriteria
- [x] De 3 passages uit het oudere bestand zijn gemerged in `lat_leespassages_leerjaar1.json` (hernummerd naar LAT-PASS-LJ1-xxx-schema om ID-conflict te vermijden)
- [x] Oud bestand verwijderd
- [x] Annotaties in gemergde passages voldoen aan hetzelfde schema als de overige (`naamval` veld bevat ook declinatie-aanduiding, zoals "nom.sg.f — 1e decl.")
- [x] Bestaande passage-tests groen
- [x] Aantal geladen passages blijft gelijk (38 totaal: 23 LAT + 15 GRC)
