# Story OL_E20_S11: Content-dekkingsrapport — script + CI-check

## Doel
Structureel zicht op content-kwaliteit per knoop. Nu moet handmatig tellen in welke mate knopen items, markdown, audio of passage-dekking hebben. Een script dat per knoop rapporteert wat ontbreekt maakt de gaps zichtbaar en voorkomt regressies wanneer nieuwe knopen worden toegevoegd.

## Input
- Alle `data/graph/*.json`
- `data/content/*.md`
- `data/audio/*.wav`
- `data/passages/*.json`

## Acceptatiecriteria
- [x] Nieuw script `scripts/content_coverage.py` produceert per knoop:
  - `has_items: bool` (len(items) > 0)
  - `has_content: bool` (`{id}.md` bestaat óf `content_ref` wijst naar bestaand bestand)
  - `has_audio: bool` (alleen V-knopen: `{id}.wav` bestaat en > 1 KB)
  - `in_passage: bool` (knoop-ID komt voor in een passage `knoop_ids`)
- [x] Samenvattings-uitvoer per type (G/V/C/I) per taal: totaal + dekkings-percentages
- [x] JSON-export optie: `--output data/content_coverage.json` voor historische vergelijking
- [x] Pytest-integratie: test faalt als LAT-G-dekking onder drempel zakt (bijv. `< 25 %`); initiële drempels conservatief zodat huidige staat slaagt
- [x] Documentatie in `scripts/README.md`
