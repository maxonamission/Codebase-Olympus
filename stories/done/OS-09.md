# Story OS-09: validation.py generiek per edge-type maken

## Doel
De cycle-detectie en topologische sortering in `graph/validation.py` generiek maken zodat ze schalen naar nieuwe edge-types zonder code-wijziging. Huidige aanpak gebruikt een hardcoded helper `_prerequisite_enrichment_subgraph` die expliciet de `transfer`-edges weghaalt; bij toevoeging van een vierde edge-type breekt die check stilletjes (nieuwe type wordt mee-gechecked op acycliciteit terwijl dat niet de bedoeling hoeft te zijn).

Afgeleid uit: graph-methodologie-reflectie (zie `docs/graph-methodology.md` § 5), waarin de per-edge-type-invariant als kern-aanbeveling staat.

## Input
- Bestaande `src/gymnasium_classica/graph/validation.py` (regel 31-44 is de hardcoded helper)
- Edge-types in Olympus: `prerequisite`, `enrichment`, `transfer` (in `models/graph.py` als `EdgeType` StrEnum)
- Bestaande tests in `tests/test_graph_validation.py` (moeten blijven slagen)

## Acceptatiecriteria
- [x] Nieuwe helper `acyclic_subgraph(graph, *, include_edge_types: set[str])` die een view teruggeeft met alleen edges van de opgegeven types
- [x] `_prerequisite_enrichment_subgraph` verwijderd; `detect_cycles` en `topological_sort` roepen `acyclic_subgraph(graph, include_edge_types={"prerequisite", "enrichment"})` aan
- [x] Module-level constant `ACYCLIC_EDGE_TYPES: frozenset[str] = frozenset({"prerequisite", "enrichment"})` zodat het beleid één plek heeft
- [x] Docstring op `ACYCLIC_EDGE_TYPES` legt uit waarom transfer níét in de set staat (bidirectioneel, cross-linguïstisch)
- [x] `docs/ONTWERPKEUZES_GYMNASIUM_CLASSICA.md` krijgt een korte sectie "Edge-type-beleid" met de regel: prerequisite + enrichment moeten acyclisch zijn, transfer mag cyclisch
- [x] Eén nieuwe test: `test_acyclic_subgraph_excludes_specified_types` met een fixture-graph die transfer-cycli bevat en verifieert dat `detect_cycles` ze niet rapporteert
- [x] Alle bestaande tests blijven groen (576+1 = 577)
- [x] `mypy src/` blijft groen (frozenset-annotatie correct)
- [x] Eindresultaat: een vierde edge-type toevoegen kost één regel — een waarde toevoegen aan of weglaten uit `ACYCLIC_EDGE_TYPES`

## Scope
Alleen `validation.py` + één test + één doc-alinea. Geen functionele gedragswijziging op de huidige dataset (prerequisite + enrichment + transfer): de output van `validate_graph()` moet identiek blijven.

## Niet in scope
- Per-edge-type orphan- of duplicate-checks (orphan en duplicate gelden over alle edges; geen aanleiding voor differentiatie)
- Configurabele edge-types via project-level config of CLI (over-engineering voor één set)
- Refactor van `topological_sort` om optioneel transfers mee te nemen (aparte story als ooit nodig)

## Aanpak
1. Schrijf `acyclic_subgraph(graph, *, include_edge_types)` — view-creatie via `graph.edge_subgraph(...)` of een `copy()` + `remove_edges_from(...)`
2. Voer `ACYCLIC_EDGE_TYPES`-constante in
3. Pas `detect_cycles` en `topological_sort` aan; verwijder de oude helper
4. Schrijf de regressietest
5. Documentatie in ONTWERPKEUZES bijwerken
6. Run `pytest`, `mypy src/`, `ruff check .` lokaal
7. Volg story-workflow: AC afvinken → naar `done/` → EPICS.md sync

## Geschat
Halve dag. Risico: NetworkX' edge-subgraph-API geeft een view, niet een copy — handlen of we mutaties verwachten (we doen alleen reads in cycle/topo, dus view is ok).

## Resultaat
- `acyclic_subgraph(graph, *, include_edge_types)` + module-constante `ACYCLIC_EDGE_TYPES = frozenset({"prerequisite", "enrichment"})` toegevoegd; `_prerequisite_enrichment_subgraph` verwijderd; `detect_cycles` en `topological_sort` filteren nu via de constante.
- Test-suite: 577 passed (576 + de nieuwe `test_acyclic_subgraph_excludes_specified_types`); `mypy src/` en `ruff check .` groen. Gedrag op de huidige dataset ongewijzigd.
- `docs/ONTWERPKEUZES_GYMNASIUM_CLASSICA.md` § "Edge-type-beleid" toegevoegd.
- **Twee kleine, bewuste afwijkingen van de letterlijke AC:**
  1. Parametertype is `frozenset[str] | set[str]` i.p.v. `set[str]`, zodat de `frozenset`-constante mypy-groen doorgegeven kan worden (set-literals in tests werken ook).
  2. De uitleg waarom `transfer` níét meedoet staat als comment-blok boven de constante; een module-level constante kan geen introspecteerbare docstring dragen, dus dit is het idiomatische equivalent. `acyclic_subgraph` geeft een `copy()` terug (zoals de oude helper), niet een view — bewust, om de originele graph niet te muteren.
