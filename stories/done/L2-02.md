# Story L2-02: Learner-model-strategie-interface + graph-aware tracing

## Doel
Abstraheer het learner model achter een **interface** zodat het huidige
BKT later vervangbaar is door PFA/logistische of graph-aware modellen,
en laat de tracing de **graph-structuur** (prerequisite- en
transfer-edges) benutten.

## Achtergrond
Ontwerpkeuze 14. In eerlijke vergelijkingen presteren PFA en — bij
grote datasets — graph-aware modellen vaak beter dan vanille-BKT, dat
skills onafhankelijk modelleert en transfer mist. Juist de combinatie
graph + per-knoop-tracing (prerequisite-driven KT) is waar de
literatuur winst ziet. BKT blijft de interpreteerbare start.

## Input
- `src/gymnasium_classica/models/learner.py` — BKT-update, `KnoopState`
- `src/gymnasium_classica/graph/loader.py` — graph + edges
  (`prerequisite`, `enrichment`, `transfer`)
- `src/gymnasium_classica/models/graph.py` — `EdgeType`,
  `PrerequisiteEdge` (encompassing_weight)

## Acceptatiecriteria
- [x] Een `LearnerModelStrategy`-interface (bijv. `predict`/`update`)
      met de huidige BKT-logica als eerste, default-implementatie
- [x] De BKT-strategie produceert identieke resultaten als de huidige
      code (geen gedragsregressie) wanneer geen graph-signaal actief is
- [x] Een graph-aware variant kan de prior/transitie van een knoop
      bijstellen op basis van de mastery van prerequisite-/transfer-buren
      (gebruikmakend van `encompassing_weight`)
- [x] De keuze van strategie is configureerbaar zonder de aanroepende
      code te wijzigen
- [x] Unit-tests: interface-contract, BKT-pariteit, en aantoonbaar
      effect van graph-aware bijstelling op een geconstrueerde mini-graph
- [x] Geen breaking changes; bestaande tests blijven groen

## Scope
Interface + BKT-pariteit + één graph-aware variant. Geen neurale/deep
knowledge tracing (expliciet buiten scope, zie uitgangspunten).

## Afhankelijkheden
- Wenselijk na L2-01 (receptief/productief), zodat de interface beide
  dimensies meeneemt.

## Geschat
Groot — raakt de kern van het learner model; pariteit goed testen.

## Resultaat
- Nieuw pakket `gymnasium_classica.learner_strategy`: abstracte
  `LearnerModelStrategy` (`predict`/`update`) + module-globale
  `get_strategy`/`set_strategy` (default `BKTStrategy`).
- `BKTStrategy`: delegeert naar de bestaande BKT-functies + praktijk-
  propagatie → **exacte pariteit** (geen gedragsregressie).
- `GraphAwareBKTStrategy`: blendt de prior van een knoop richting een met
  `encompassing_weight` gewogen gemiddelde van prerequisite-/transfer-buren
  vóór de BKT-update (prerequisite-driven KT). Zonder buren identiek aan BKT.
- `scheduling/session.py::_process_response` routeert nu via
  `get_strategy().update(...)` — strategie wisselbaar zonder de callsite te
  raken. Neemt de L2-01-richting mee.
- Tests: interface-contract, BKT-pariteit (posterior + propagatie identiek),
  graph-aware optrekt-effect op mini-graph, no-neighbours == BKT, validatie.
  **630 tests groen**, geen regressie. Neural/deep KT buiten scope.
