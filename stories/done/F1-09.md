# Story F1-09: Ontsluit of verwijder vocabulaire_clusters.json

## Doel
`data/vocabulaire_clusters.json` bevat 13 semantische clusters (familie, oorlog, lichaam, politiek, etc.) met beschrijvingen, maar wordt nergens in runtime-code gebruikt. Vocab-knopen in `vocab_sources/*` hebben wel een `cl`-veld dat naar deze labels verwijst. Twee opties: ontsluiten in de frontend (thema-filter / learning-focus), of verwijderen als niet-gebruikte data.

## Input
- `data/vocabulaire_clusters.json`
- `data/vocab_sources/*.json` (`cl`-veld)
- F1-05 (woordkaart) — logische plek om cluster-label te tonen

## Acceptatiecriteria
- [x] Beslissing gedocumenteerd: ontsluiten óf verwijderen. Beide zijn acceptabele uitkomsten.
- [x] Bij ontsluiten:
  - Clusters beschikbaar via `/progress/clusters` endpoint (lijst + voortgang per cluster)
  - Frontend dashboard toont cluster-voortgang (% V-knopen in cluster beheerst)
  - Optioneel: "focus-sessie" waarin scheduling één cluster prioriteert
- [x] Bij verwijderen:
  - Bestand verplaatsen naar `docs/archief/` of verwijderen
  - `cl`-velden in vocab_sources laten staan (zelfbeschrijvend)
  - README vermelden
