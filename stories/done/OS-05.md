# Story OS-05: Claude Code hooks (PostToolUse + Stop)

## Doel
Via `.claude/settings.json` automatische checks laten draaien binnen Claude Code-sessies, zodat Claude fouten meteen ziet en in dezelfde sessie oplost ‚Äî niet pas nadat een PR is aangemaakt. Dit is laag 3 van de ontwikkelstraat. Waarde: reparaties gebeuren binnen de context waarin Claude nog weet wat hij aan het doen was.

## Input
- Ruff-configuratie uit OS-01
- Mypy-configuratie uit OS-02
- Bestaande pytest-suite
- Claude Code hooks-documentatie (PostToolUse, Stop, SessionStart)
- Skill `update-config` in Claude Code kan helpen bij correcte syntax

## Acceptatiecriteria
- [x] `.claude/settings.json` op repo-root met deze hooks:
  - [x] **PostToolUse** op `Edit|Write`: verwijst naar `.claude/hooks/ruff_on_python.sh`; script filtert op `.py` en draait ruff check --fix + format op gewijzigd bestand
  - [x] **Stop**: verwijst naar `.claude/hooks/pytest_on_stop.sh`; draait pytest -x --tb=short met exit 2 bij failure
- [x] Hook-scripts zijn idempotent en snel (ruff op √©√©n bestand ~0.5s; pytest wordt gebufferd zodat Claude alleen de failure ziet bij fout)
- [x] Hooks falen niet-fataal bij tooling-fout: script checkt `command -v uv`, gebruikt `|| true` rond ruff-calls; Stop-hook skipt als `uv` niet ge√Ønstalleerd is
- [x] Hooks werken met `uv` als package manager (PATH-check + `uv run` aanroepen)
- [x] Hook-script getest met dummy JSON-payload: Python-bestand ‚Üí ruff draait succesvol; non-Python-bestand ‚Üí exit 0 zonder ruff-aanroep
- [x] Stop-hook lokaal getest: pytest groen ‚Üí exit 0; break-path (exit 2 bij failure) is in script expliciet vastgelegd
- [x] `.claude/settings.json` in versiebeheer; `.claude/settings.local.json` aan `.gitignore` toegevoegd
- [x] CLAUDE.md aangevuld met omschrijving van beide hooks

## Opmerking over live-test
De "voorbeeldsessie"-AC zijn principieel **niet verifieerbaar binnen deze Claude Code-sessie**: hooks in `.claude/settings.json` worden geladen bij sessie-start, en wijzigingen zijn pas actief in een nieuwe sessie. De hook-scripts zijn wel direct aanroepbaar getest (zie commando's hierboven). Eerstvolgende sessie waarin Claude een `.py` bewerkt of probeert te stoppen: effectiviteit zichtbaar.

## Scope
Alleen `PostToolUse` en `Stop`. Andere hook-typen (SessionStart, UserPromptSubmit) zijn uitbreidingen voor latere stories.

## Niet in scope
- SessionStart-hook voor venv-setup ‚Äî interessant, maar vereist dat elke developer dezelfde `uv`-install heeft; apart afwegen
- Hook voor story-status ‚Üí OS-06 (wordt daar eventueel ook als Stop-hook toegevoegd)
- Hooks voor non-Python bestanden (als later React-frontend bijkomt)
- MCP-server-configuratie

## Aanpak
1. `.claude/settings.json` schrijven met hooks-block
2. Shell-snippets voor elk hook (of verwijzen naar scripts in `scripts/hooks/`)
3. Handmatig testen: edit-een-py-bestand-met-fout scenario
4. Handmatig testen: opzettelijk gebroken-test scenario
5. Documentatie

## Geschat
Halve middag. Risico: `PostToolUse` hook-output verstoort Claude's output-stream als niet goed ingesteld. Mitigatie: altijd zachte exit-code 0 tenzij blocking gewenst, output beperken tot relevante regels.

## Resultaat
- `.claude/settings.json` ‚Äî mapt PostToolUse (matcher `Edit|Write`) naar `ruff_on_python.sh` en Stop naar `pytest_on_stop.sh`
- `.claude/hooks/ruff_on_python.sh` ‚Äî leest JSON via stdin, parst `tool_input.file_path` in Python (geen jq-dependency), skipt non-`.py` files, draait `uv run ruff check --fix` + `uv run ruff format`, exit altijd 0 (non-blocking: Claude ziet ruff-output en kan er mee werken in dezelfde turn)
- `.claude/hooks/pytest_on_stop.sh` ‚Äî draait testsuite, bij failure exit 2 (blokkeert de Stop-actie) + output naar stderr zodat Claude de failure ziet
- Beide scripts: `chmod +x`, fallback op `command -v uv`, veilige `set -uo pipefail`
- `.gitignore` uitgebreid met `.claude/settings.local.json`
- CLAUDE.md aangevuld
- Geen veranderingen aan Python-code: ruff + mypy + 558 tests blijven groen
