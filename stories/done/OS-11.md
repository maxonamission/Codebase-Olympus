# Story OS-11: Adopteer codebase-standards canoniek story-format + gedeelde drift-validator

## Doel
Olympus' story-administratie gelijktrekken met de gedeelde `codebase-standards`
(v0.1.0): front-matter `status:` als bron van waarheid, een gedeelde
ID-schema-agnostische drift-validator, en een canonieke Statustabel +
`PROJECTSTATUS.md` met `done/total`-tellingen. IDs blijven het vakinhoudelijke
schema (`A1-01`, `B4-02`); hernummering is bewust uitgesteld (gefaseerde adoptie).

## Input / afhankelijkheden
- `codebase-standards` v0.1.0 (gedeelde scripts + canoniek format).
- Bestaande Olympus-ontwikkelstraat (OS-00 t/m OS-09): ruff, mypy, pre-commit, CI.

## Acceptatiecriteria
- [x] Alle stories hebben YAML front-matter (`type/project/epic/story_id/status/prioriteit`); `status:` is de bron, map wordt erop gesynct.
- [x] Gevendorde gedeelde scripts in `scripts/`: `check_story_status.py`, `sync_story_folders.py`, `regenerate_status.py`, `check_standard_drift.py` (byte-identiek aan upstream; uit lokale lint/format gehouden).
- [x] `stories/EPICS.md` heeft een canonieke Statustabel met `done/total` per epic + totaalregel; `PROJECTSTATUS.md` aangemaakt met epic-dashboard + `done/total`.
- [x] Gedeelde validator + regenerator + drift-check in pre-commit én CI; `--ac-gate=warn` tijdens legacy-opruiming.
- [x] Adoptie-stempels: `.codebase-standards-version` (0.1.0), `.codebase-standards-manifest.json`, `.codebase-standards-sync.json`.
- [x] `uv run python scripts/check_story_status.py --mode=full --ac-gate=warn`, `regenerate_status.py --check`, ruff, mypy en pytest groen.

## Scope
Binnen: front-matter-retrofit (199 stories), vendoren, EPICS/PROJECTSTATUS naar
canoniek format, CI/pre-commit-wiring, CLAUDE.md-werkwijze bijwerken.
Buiten: hernummering naar `E#_S#` (aparte, nog niet ingeplande story); de
AC-gate van `warn` naar `error` tillen (vereist legacy-AC-opschoning).

## Resultaat
- 199 stories voorzien van front-matter (182 done, 1 doing, 16 backlog), 30
  materialized epics in de Statustabel.
- De gedeelde validator is upstream ID-schema-agnostisch gemaakt (classificatie
  op front-matter i.p.v. een vast `E#_S#`-patroon) zodat Olympus' IDs zonder
  hernummering passen — die generalisatie is teruggevloeid in `codebase-standards`.
- Validator (full, ac-gate=warn): 0 fouten / 0 waarschuwingen. Drift-checks groen.
  mypy: 0 issues. pytest: 779 passed.
